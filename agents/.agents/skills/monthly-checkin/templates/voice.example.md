# Check-in voice

Customize this file from the user's real edits and published writing.

## Core voice

- Lead with a claim, not a resume framing.
- State what changed. Let evidence demonstrate value.
- Prefer concrete consequences over abstractions.
- Use technical detail as proof, not as the subject.
- Credit collaborators through complementary ownership.

## Avoid

- vague performance-review filler;
- PR-by-PR recaps;
- unsupported claims;
- treating routine review as co-ownership;
- growth sections that only list future deliverables.

## Punctuation and rhythm

Record the user's punctuation preferences, sentence rhythm, and a few non-confidential before/after examples.
